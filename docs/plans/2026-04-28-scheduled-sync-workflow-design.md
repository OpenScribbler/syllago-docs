# Scheduled Sync Workflow — Design

**Date:** 2026-04-28
**Scope:** Cross-repo automation: `OpenScribbler/syllago-docs` ↔ `OpenScribbler/syllago`
**Author:** Holden + Maive

## Problem

`syllago-docs` carries auto-generated content (CLI reference, capability matrices, canonical-keys, provider data, sidebar managed-blocks) that's regenerated by `bun run sync` from JSON manifests in syllago's GitHub release assets. The script is wired as `prebuild`, so CI deploys always reflect the latest upstream — but the **tracked source files only update when a human runs `bun run sync` and commits**. That divergence shipped a deploy break on 2026-04-28: `syllago create` was removed upstream, prebuild deleted the auto-generated reference page in CI, but the repo still tracked a stale `cli-reference/create.mdx`. Contributors edited narrative pages assuming the page existed → broken links shipped to main when `starlight-links-validator` tightened (PRs #30, #31, #32 were the recovery chain).

## Goal

Detect upstream drift on a schedule, surface it as a PR pre-merge, and fail the existing PR-side link-validator gate instead of post-merge deploy. Never accumulate more than a few days of staleness.

## Cadence reality

syllago releases are bursty:
- v0.9.0 → v0.10.0: 8 days
- v0.10.0 → v0.10.3: 4 releases in ~45 hours

Pure cron either misses bursts (daily skips 3 of 4) or generates noise in lulls (8 cosmetic-only PRs between releases). Trigger has to track the actual release event.

## Design decisions

### D1 — Trigger: `repository_dispatch` from syllago's `release.yml` + weekly cron backstop + manual dispatch

- **Primary:** syllago's `release.yml` final step fires `repository_dispatch` with `event_type: syllago-release` and `client_payload: { tag: vX.Y.Z }`. Latency: minutes from release. Catches every release exactly once.
- **Backstop:** weekly cron (Mondays 14:00 UTC) catches PAT expiry, dispatch step regression, or any non-tag-driven manifest changes.
- **Manual:** `workflow_dispatch` for ad-hoc runs.
- **Cross-repo auth:** PAT scoped to `syllago-docs:contents:write,actions:write`, stored in syllago as `SYLLAGO_DOCS_DISPATCH_PAT`. (Considered: GitHub App. Rejected for v1 — overkill for one event type, PAT rotation cost is real but bounded.)

### D2 — No-drift behavior: skip silently

After `bun run sync`, run `git diff --exit-code src/`. If clean → write `## Sync status: in sync with syllago vX.Y.Z` to `$GITHUB_STEP_SUMMARY` and exit 0. No PR, no comment, no issue. The runlog is the artifact. Sync scripts already only `writeFile` when content changes (cerebrum learning), so the diff check is reliable.

### D3 — On drift detection: open a normal PR, let `lint.yml` gate it

The bot's PR runs through `lint.yml`'s `Build` job (added in PR #31) like any other PR. The Build step runs `bun run build` which runs `starlight-links-validator`. If sync output breaks links:
- PR shows red CI.
- Human commits narrative-page fixes to the rolling branch.
- CI re-runs green, merge.

**No tracking issue, no draft state.** The PR itself is the surface. The sync workflow attempts a local build before opening the PR (to populate the diff classifier and surface MDX parse errors), but opens the PR regardless of build outcome — a red-build PR is more useful than no PR for fix-forward.

### D4 — Diff classification: cosmetic vs. material

PR body categorizes the diff so review effort scales with material change:

```
Sync from syllago vX.Y.Z (was vA.B.C)

Cosmetic (version stamp / generatedAt timestamp): N files
Material:
  - <list of paths from the classifier>
```

Classifier rule: a file is **cosmetic** iff every changed line matches a regex allowlist (timestamp regex, `Generated from syllago` regex, `generatedAt` JSON field). Anything else → **material**. When Material is empty, the user merges in 5 seconds.

**No auto-merge in v1.** CI green here means link-validator + Astro build, which doesn't catch silent regressions (canonical-key page that came back empty, sidebar entry that dropped a content type, matrix row removed by upstream filter changes). 30 seconds of human eyeballing is the right cost. Auto-merge revisits when the categorizer has a track record.

### D5 — Branch + PR lifecycle: rolling `chore/sync-upstream`, force-push, single open PR

- Branch: `chore/sync-upstream` (no version in branch name).
- Force-push on every run.
- One PR open at a time, title regenerated per run: `chore(sync): regenerate auto-content against syllago vX.Y.Z`.
- `delete-branch-on-merge` (already enabled per CHANGELOG 2026-04-24) cleans up post-merge.

Versioned branches collapse under burst cadence (4 PRs in 45 hours during 0.10.0 → 0.10.3). Rolling branch always represents *current* drift from main, which is the only state that matters.

**Force-push caveat:** if the existing PR has unresolved review comments, post a comment instead of force-pushing. Probably overengineering for v1; revisit if reviews routinely span multiple syncs.

### D6 — Build vs. targeted link-check: full build

Run `bun astro build` (skipping the redundant `prebuild` hook since we just ran sync). Full build catches:
- MDX parse errors from upstream content (2026-04-25 `<url>` autolink, 2026-04-22 `moat-001.mdx` angle-bracket).
- Astro content-collection schema drift (new field arrives in `capabilities.json` that `content.config.ts` rejects).
- llms.txt postbuild failures.

Cost: ~3-5 min on cache hit, run 2-4×/week peak. CI minutes not a constraint at this volume.

### D7 — Concurrency: cancel-in-progress

```yaml
concurrency:
  group: sync-upstream
  cancel-in-progress: true
```

Burst case (3 dispatches in 60s): only the latest tag wins, earlier runs cancel. End state: one run, one PR, against the latest tag. Distinct group from `lint.yml` and `deploy.yml` so cross-cancellation can't happen.

### D8 — CHANGELOG: auto-generated entry per sync

The repo's hookify rule requires a CHANGELOG entry when files under `src/content/docs/` change. Bot's commit must include one or it gets blocked.

Generator inserts:

```markdown
### Synced (syllago vX.Y.Z)
- Cosmetic regen: <N> files (<short summary, e.g., canonical-keys version stamps, data-quality timestamps>).
- Material: <list of paths from D4 classifier>.
```

Insertion logic: if `## YYYY-MM-DD` heading exists for today, append `### Synced` block under it; otherwise prepend a new dated section above the most recent one. Format mirrors the 2026-04-28 entry's `### Synced (syllago 0.10.0 → 0.10.3)` subsection.

### D9 — Vouch the bot

`syllago/.github/VOUCHED.td` enforces that PR authors are vouched (via `pr-policy.yml` on syllago-docs, configured with `vouched-repo: OpenScribbler/syllago`). `dependabot[bot]` is already there. Add `github-actions[bot]` (alphabetical insert between `dependabot[bot]` and `holdenhewett`).

The actor is `github-actions[bot]` because the sync workflow lives inside `syllago-docs` and creates a same-repo PR using `secrets.GITHUB_TOKEN`. The cross-repo PAT is only for the dispatch direction; the PR author identity is `github-actions[bot]`.

## Workflow steps (sync-upstream.yml)

```
1. validate-actions (mirrors deploy.yml/lint.yml convention)
2. checkout, setup Node/bun, install deps, install D2 (mirrors deploy.yml)
3. bun run sync
4. git diff --exit-code src/
   → if clean: write step summary, exit 0
5. classify diff (cosmetic vs. material)
6. generate CHANGELOG entry, prepend to CHANGELOG.md
7. bun astro build (capture exit code, don't fail yet)
8. force-push to chore/sync-upstream
9. create-or-update PR
   - title: chore(sync): regenerate auto-content against syllago vX.Y.Z
   - body: classified diff + (if step 7 failed) ⚠ build-failed warning
10. exit with step 7's code
```

PR-side gate (already in place via `lint.yml`'s `Build` job): runs `bun run build` on the bot's PR including link validation. Red CI → human commits fixes to rolling branch. Green → merge.

## Cross-repo prerequisites (must land before workflow goes live)

1. **syllago-docs side:** workflow file, classifier script, CHANGELOG generator. Test via `workflow_dispatch` before the auto-trigger lands.
2. **syllago side:**
   - Vouch `github-actions[bot]` in `.github/VOUCHED.td`.
   - Add `SYLLAGO_DOCS_DISPATCH_PAT` repo secret (fine-grained PAT scoped to `syllago-docs:contents:write,actions:write`). Note expiry for rotation.
   - Add dispatch step to `release.yml` final block (one `gh api` or `actions/github-script` call after homebrew formula update).

## Rollback

Disable by removing the dispatch step in syllago `release.yml` and disabling the cron schedule in syllago-docs `sync-upstream.yml`. The sync workflow file can stay in place for `workflow_dispatch`-only operation. No state to clean up — the rolling branch self-deletes on next merge or manual delete.

## Open questions deferred to v2

- Auto-merge for cosmetic-only diffs. Defer until classifier has a multi-week track record of correct cosmetic/material calls.
- Force-push protection when reviews are pending. Defer until reviews routinely span multiple syncs.
- Multi-tag dispatch handling (e.g., re-tag of an existing release). Current design assumes monotonic tag sequence; concurrency group covers the burst case.
