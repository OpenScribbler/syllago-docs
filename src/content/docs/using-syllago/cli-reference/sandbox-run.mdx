title: "syllago sandbox run"
description: "Run a provider in a sandbox"
---

## Synopsis

```
syllago sandbox run <provider> [flags]
```

## Options

| Flag | Type | Default | Required | Description |
|------|------|---------|----------|-------------|
| `--allow-domain` | `stringArray` | тАФ | No | Allow an additional domain for this session |
| `--allow-env` | `stringArray` | тАФ | No | Forward an additional env var into the sandbox |
| `--allow-port` | `stringArray` | тАФ | No | Allow a localhost port inside the sandbox |
| `--force-dir` | `bool` | тАФ | No | Skip directory safety checks |
| `--mount-ro` | `stringArray` | тАФ | No | Mount additional path read-only inside sandbox |
| `--no-network` | `bool` | тАФ | No | Block all network egress (no proxy) |

## Global Options

| Flag | Type | Default | Required | Description |
|------|------|---------|----------|-------------|
| `--json` | `bool` | тАФ | No | Output in JSON format |
| `--no-color` | `bool` | тАФ | No | Disable color output |
| `-q`, `--quiet` | `bool` | тАФ | No | Suppress non-essential output |
| `-v`, `--verbose` | `bool` | тАФ | No | Verbose output |

## Examples

```bash
  # Run Claude Code in a sandbox
  syllago sandbox run claude-code

  # Run with additional domain access
  syllago sandbox run claude-code --allow-domain api.example.com

  # Run with no network at all
  syllago sandbox run gemini-cli --no-network
```
