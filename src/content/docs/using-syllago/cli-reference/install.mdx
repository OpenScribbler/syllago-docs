## Synopsis

```
syllago install [name] --to <provider>
```

## Description

Install content from your library into a provider's location.

By default uses a symlink so edits in your library are reflected immediately.
Use `--method copy` to place a standalone copy instead.

Examples:
```sh
# Install a skill to Claude Code
syllago install my-skill --to claude-code

# Install with a standalone copy instead of symlink
syllago install my-skill --to cursor --method copy

# Install all skills to a provider
syllago install --to claude-code --type skills

# Preview what would happen
syllago install my-skill --to claude-code --dry-run
```

## Options

| Flag | Type | Default | Required | Description |
|------|------|---------|----------|-------------|
| `--to` | `string` | ‚Äî | Yes | Provider to install into |
| `--type` | `string` | ‚Äî | No | Filter to a specific content type |
| `--method` | `string` | `symlink` | No | Install method: `symlink` (default) or `copy` |
| `-n`, `--dry-run` | `bool` | ‚Äî | No | Show what would happen without making changes |
| `--base-dir` | `string` | ‚Äî | No | Override base directory for content installation |
| `--no-input` | `bool` | ‚Äî | No | Disable interactive prompts, use defaults |

## Portability Warnings

When installing content, syllago runs the format converter and surfaces **portability warnings** if the content uses features unsupported by the target provider. Warnings appear inline in text mode and in the `warnings` array in JSON mode.

## JSON Output

```json
{
  "installed": [
    {
      "name": "my-skill",
      "type": "skills",
      "method": "symlink",
      "path": "/home/user/.claude/skills/my-skill",
      "warnings": ["Portability warning 1"]
    }
  ],
  "skipped": [
    { "name": "skipped-item", "reason": "error message" }
  ]
}
```
