---
title: "syllago registry"
description: "Manage git-based content registries"
---

Add, remove, list, and sync git repositories as content registries.

Registries are read-only git repos containing shared content (skills, rules,
hooks, MCP configs, etc.). Use "registry sync" to pull updates, and
"registry items" to browse what's available.

To use registry content, browse it in the TUI ("syllago") or install it
directly with "syllago install --to &lt;provider&gt;".

## Subcommands

| Command | Description |
|---------|-------------|
| [`add`](/syllago-docs/using-syllago/cli-reference/registry-add/) | Add a git registry |
| [`create`](/syllago-docs/using-syllago/cli-reference/registry-create/) | Create a new registry |
| [`items`](/syllago-docs/using-syllago/cli-reference/registry-items/) | Browse content available in registries |
| [`list`](/syllago-docs/using-syllago/cli-reference/registry-list/) | List registered registries |
| [`remove`](/syllago-docs/using-syllago/cli-reference/registry-remove/) | Remove a registry and delete its local clone |
| [`sync`](/syllago-docs/using-syllago/cli-reference/registry-sync/) | Pull latest content from one or all registries |
