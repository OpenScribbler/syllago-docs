## Synopsis

```
syllago convert <name> --to <provider>
```

## Description

Renders a library item to a target provider's format without installing it.
Output goes to stdout by default, or to a file with `--output`.

No state changes are made — this is purely for ad-hoc sharing.

Examples:
```sh
# Convert a skill to Cursor format (stdout)
syllago convert my-skill --to cursor

# Convert and save to a file
syllago convert my-rule --to windsurf --output ./windsurf-rule.md

# Convert to JSON output
syllago convert my-skill --to cursor --json
```

## Options

| Flag | Type | Default | Required | Description |
|------|------|---------|----------|-------------|
| `--to` | `string` | — | Yes | Target provider |
| `-o`, `--output` | `string` | — | No | Write output to this file path (default: stdout) |
| `--batch` | `string` | — | No | Directory of hook files to batch-canonicalize (mutually exclusive with positional name) |
| `--from` | `string` | — | No | Source provider slug (required with `--batch`) |
| `--dry-run` | `bool` | — | No | Show what would be converted without writing files |

## Portability Warnings

When converting content, syllago surfaces **portability warnings** if the content uses features unsupported by the target provider. Warnings appear on stderr in text mode and in the `warnings` array in JSON mode.

## JSON Output

```json
{
  "name": "my-skill",
  "provider": "cursor",
  "output": "path/to/output.md",
  "warnings": ["Warning message 1", "Warning message 2"]
}
```

## Global Options

| Flag | Type | Default | Required | Description |
|------|------|---------|----------|-------------|
